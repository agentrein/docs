---
title: wrap()
description: Wrap any third-party client with automatic rollback protection — zero changes to your existing code.
---

## Overview

`wrap()` is the easiest way to protect your agent's API calls. Pass your existing client and get back a Proxy that looks and feels identical — but every method call is automatically logged and rolled back on failure.
```ts
const agentrein = new AgentRein({ apiKey: process.env.AGENTREIN_API_KEY });
const session = await agentrein.newSession({
  agentId: 'my-agent',
  intent: 'Onboard new customer',
});

const agentOctokit = agentrein.wrap(octokit, session, { connector: 'github' });
const agentSlack = agentrein.wrap(slackClient, session, { connector: 'slack' });
const agentStripe = agentrein.wrap(stripe, session, { connector: 'stripe' });

// Use exactly as before — nothing changes
await agentOctokit.issues.create({ owner, repo, title });
await agentSlack.chat.postMessage({ channel, text });
await agentStripe.invoices.create({ customer: 'cus_xxx' });

await agentrein.completeSession(session);
```

## Signature
```ts
agentrein.wrap<T extends object>(
  client: T,
  session: Session,
  options: WrapOptions
): T
```

The return type is always `T` — the same type as the input client. TypeScript autocomplete and type checking are fully preserved.

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `client` | `T` | Yes | Any third-party client instance (Octokit, WebClient, Stripe, etc.) |
| `session` | `Session` | Yes | The session returned by `newSession()` |
| `options.connector` | `string` | Yes | Connector prefix used to build the `apiName` e.g. `'github'`, `'slack'`, `'stripe'` |

## How It Works

1. Every method call on the returned proxy is intercepted
2. `apiName` is built from `connector + nested property path` e.g. `github.issues.create`
3. The original function executes with correct `this` binding
4. The action is logged to the backend asynchronously (fire-and-forget)
5. If the call throws — rollback is triggered automatically for the session

## Supported Connector Prefixes

These prefixes map to built-in rollback strategies in the AgentRein backend:

| Prefix | Client |
|---|---|
| `github` | `Octokit` |
| `slack` | `WebClient` from `@slack/web-api` |
| `stripe` | `Stripe` |
| `hubspot` | HubSpot API client |
| `salesforce` | Salesforce API client |
| `notion` | Notion API client |
| `gmail` | Google Gmail client |
| `gdrive` | Google Drive client |
| `gsheets` | Google Sheets client |

<Info>
Using a prefix not in the list above will still log the action, but no built-in rollback strategy will be found. The action will be marked `ROLLBACK_FAILED` if rollback is triggered. Use `call()` with a custom `undoConfig` for unsupported connectors.
</Info>

## Nested Clients

`wrap()` handles arbitrarily nested method paths automatically:
```ts
// 2 levels
await agentSlack.chat.postMessage({ channel, text });
// apiName → slack.chat.postMessage

// 3 levels
await agentOctokit.rest.issues.create({ owner, repo, title });
// apiName → github.rest.issues.create
```

## Fail-Open Guarantee

If the AgentRein backend is unreachable, `wrap()` still executes the original method and returns the result. Logging and rollback are silently skipped. Set `failureMode: 'closed'` on the `AgentRein` constructor to disable this.
