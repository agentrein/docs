---
title: call()
description: Wrap an agent action with automatic rollback protection.
---

## Signature
```ts
agentrein.call(fn, session, ...args): Promise<T>
agentrein.call(fn, session, undoConfig, ...args): Promise<T>
```

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `fn` | `Function` | Yes | The action to execute. Pass a plain async function. |
| `session` | `Session` | Yes | The session returned by `newSession()`. |
| `undoConfig` | `UndoConfig` | No | Custom rollback instructions. Created with `createUndoConfig()`. |
| `...args` | `any[]` | No | Arguments forwarded to `fn`. Last arg may be `CallOptions`. |

## CallOptions

| Option | Type | Default | Description |
|---|---|---|---|
| `actionName` | `string` | `fn.name` | Override the logged action name. Required when passing a plain function. |
| `requiresApproval` | `boolean` | `false` | If true, action is held for human approval before executing. |
| `pollIntervalMs` | `number` | `2000` | How often to poll for approval decision. |
| `timeoutMs` | `number` | `86400000` | How long to wait for approval before throwing. |

<Warning>
The SDK detects CallOptions by checking if the last argument contains the key `actionName` or `requiresApproval`. An API payload that contains either of these keys will be misinterpreted as CallOptions. If your payload uses these field names, use the `undoConfig` parameter signature to separate options from args: `agentrein.call(fn, session, createUndoConfig({type: "none"}), yourPayload, callOptions)`

Only `actionName` and `requiresApproval` are used to detect CallOptions. Passing an object with only `pollIntervalMs` or `timeoutMs` as the last arg will NOT be interpreted as CallOptions.
</Warning>

## Behavior

1. Executes `fn`
2. Logs the action to the backend asynchronously (fire-and-forget)
3. If `fn` throws — triggers `POST /sessions/:id/rollback` before re-throwing

## With a Built-in Connector

> [!NOTE]
> When using a ConnectorAction, all args are passed as a single object to the connector's execute() method. Multiple positional args after the undoConfig are not supported for Connector Actions - use a single args object.

```ts
import Stripe from 'stripe';
const stripe = new Stripe(process.env.STRIPE_SECRET_KEY);

const invoice = await agentrein.call(
  stripe.invoices.create,
  session,
  { customer: 'cus_xxx', currency: 'usd' },
  { actionName: 'stripe.invoices.create' }
);
```

Explicitly showing that the payload object and options are separate last args:

```ts
const invoiceWithOptions = await agentrein.call(
  stripe.invoices.create,
  session,
  { customer: 'cus_xxx', currency: 'usd' },
  { actionName: 'stripe.invoices.create', requiresApproval: false }
);
```

## With a Plain Function
```ts
const result = await agentrein.call(
  myApiFunction,
  session,
  { actionName: 'myConnector.resource.create' }
);
```

## Fail-Open Guarantee

If the AgentRein backend is unreachable, `call()` still executes `fn` and returns the result. Logging and rollback are silently skipped. Set `failureMode: 'closed'` to disable this.
