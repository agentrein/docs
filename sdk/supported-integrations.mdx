## Credentials

Add connector credentials in **Dashboard → Integrations**. Each credential
has a Key (the token name below) and a Value (Plain Text or JSON).

<Tabs>
  <Tab title="GitHub" icon="github">
    **Key:** `GITHUB_TOKEN`
    **Type:** Plain Text

    Get your token at: [GitHub → Settings → Developer settings → Personal access tokens](https://github.com/settings/tokens)
  </Tab>

  <Tab title="Slack">
    **Key:** `SLACK_TOKEN`
    **Type:** Plain Text

    Get your token at: [Slack API → Your Apps → OAuth & Permissions → Bot Token](https://api.slack.com/apps)
  </Tab>

  <Tab title="Stripe">
    **Key:** `STRIPE_TOKEN`
    **Type:** Plain Text

  </Tab>

  <Tab title="HubSpot">

    **Type:** Plain Text

  </Tab>

  <Tab title="Salesforce">

    **Required scope:** `https://www.googleapis.com/auth/gmail.modify`

    <Steps>

      </Step>
      <Step title="Generate refresh token">

      </Step>
    </Steps>
  </Tab>

## How Rollback Works

<Tip>
When you use `wrap()`, AgentRein intercepts every method call and logs it
to the backend with the action name (e.g. `stripe.invoices.create`) and
the method response. The backend maintains a **Rollback Registry** — a map
of action names to compensation strategies.

When rollback is triggered for a session, the backend looks up each action
in the registry and executes the appropriate undo strategy in LIFO order.
</Tip>

## Registry

The following action names have automatic rollback strategies registered
in the backend:

<Tabs>
  <Tab title="GitHub" icon="github">
    | Action | Undo Strategy |
    |---|---|
    | `github.issues.create` | Close the issue |
    | `github.issues.update` | Restore `beforeState` |
  </Tab>
  <Tab title="Slack">
    | Action | Undo Strategy |
    |---|---|
    | `slack.chat.postMessage` | Updates the message to a Tombstone (⚠️ Reverted by AgentRein: [reason]) and clears all blocks/buttons |
    | `slack.chat.update` | Restore `beforeState` |
    | `slack.chat.delete` | Re-posts the original message content from beforeState with a restoration notice |
  </Tab>
  <Tab title="Stripe">
    | Action | Undo Strategy |
    |---|---|
    | `stripe.invoices.create` | Delete the invoice |
    | `stripe.customers.create` | Delete the customer |
    | `stripe.refunds.create` | No undo (logged only) |
  </Tab>
  <Tab title="HubSpot">
    | Action | Undo Strategy |
    |---|---|
    | `hubspot.contacts.create` | Delete the contact |
    | `hubspot.contacts.update` | Restore `beforeState` |
    | `hubspot.deals.create` | Delete the deal |
    | `hubspot.deals.update` | Restore `beforeState` |
  </Tab>
  <Tab title="Salesforce">
    | Action | Undo Strategy |
    |---|---|
    | `salesforce.contacts.create` | Delete the record |
    | `salesforce.contacts.update` | Restore `beforeState` |
    | `salesforce.opportunities.create` | Delete the record |
    | `salesforce.opportunities.update` | Restore `beforeState` |
  </Tab>
  <Tab title="Notion">
    | Action | Undo Strategy |
    |---|---|
    | `notion.pages.create` | Move to trash |
    | `notion.pages.update` | Restore `beforeState` |
    | `notion.database_items.create` | Move to trash |
    | `notion.blocks.append` | Delete appended blocks |
  </Tab>
  <Tab title="Gmail">
    | Action | Undo Strategy |
    |---|---|
    | `gmail.messages.send` | Send correction reply in thread |
    | `gmail.messages.trash` | Untrash the message |
    | `gmail.drafts.create` | Delete the draft |
    | `gmail.labels.modify` | Reverse the label changes |
  </Tab>
  <Tab title="Google Drive">
    | Action | Undo Strategy |
    |---|---|
    | `gdrive.files.create` | Move the file to trash (soft delete — recoverable for 30 days) |
    | `gdrive.files.update` | Restore `beforeState` |
    | `gdrive.files.move` | Reverse the move |
    | `gdrive.files.trash` | Untrash the file |
  </Tab>
  <Tab title="Google Sheets">
    | Action | Undo Strategy |
    |---|---|
    | `gsheets.values.append` | Clear the appended range |
    | `gsheets.values.update` | Restore `beforeState` |
    | `gsheets.spreadsheets.create` | Move the spreadsheet to trash (soft delete — recoverable for 30 days) |
    | `gsheets.sheets.add` | Delete the added sheet |
  </Tab>
</Tabs>

## Validating Credentials Before Execution

Before running your agent, you can verify that all required connector credentials
are configured in the dashboard using `validateConfig()`:

```ts
const result = await agentrein.validateConfig(['github', 'slack']);

if (!result.valid) {
  console.error('Missing credentials:', result.missing);
  // result.suggestions → ['Configure GitHub credentials at Dashboard → Settings → Connectors']
  process.exit(1);
}
```

### Return Value

| Field | Type | Description |
|---|---|---|
| `valid` | `boolean` | `true` only if ALL requested connectors have credentials configured |
| `configured` | `string[]` | Connectors that have credentials stored |
| `missing` | `string[]` | Connectors that do not have credentials stored |
| `suggestions` | `string[]` | Human-readable setup instructions for each missing connector |

<Warning>
Rollback will fail with `ROLLBACK_FAILED` if connector credentials are not 
configured before executing actions. Always call `validateConfig()` during 
your agent's startup to catch this early.
</Warning>

<Warning>
## Unsupported Services

For services not in the registry, `wrap()` will still log all actions and 
trigger session rollback on failure — but the compensation step will be 
marked `ROLLBACK_FAILED` since no undo strategy exists yet.

To request support for a new connector, open an issue at 
[github.com/AgentReinAi](https://github.com/AgentReinAi).
</Warning>
