# Connectors

The AgentRein SDK does **NOT** provide connector wrappers. Instead of learning a new abstraction layer, developers bring their own standard API clients (Stripe, Slack, GitHub, etc.).

## Server-Side Rollback Strategies

The server-side backend has built-in rollback strategies for Stripe, Slack, GitHub, and other platforms. These rollback actions run **automatically on the AgentRein server** during a rollback, not in the developer's app. 

Because the rollback executes on the server infrastructure, your backend service requires the necessary secret API keys (e.g., `STRIPE_KEY`, `SLACK_TOKEN`), rather than managing connection secrets inside the SDK environment.

## Usage Pattern

To use a connector with AgentRein, wrap the native API client call in `agentrein.call()`.

```javascript
import Stripe from 'stripe';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY);

await agentrein.call(
  stripe.invoices.create, 
  session, 
  { customer: 'cus_xxx' }, 
  { actionName: 'stripe.invoices.create' }
);
```

### Required `actionName` Format

Notice the `actionName` argument. It should always follow the pattern `'connector.resource.operation'` (e.g., `'stripe.invoices.create'`). The AgentRein backend parses this string to know which specific rollback strategy to apply automatically.

## Support Matrix

| Connector     | Action name to use              | Auto-rollback on server | What the server does to undo                      |
|---------------|---------------------------------|-------------------------|---------------------------------------------------|
| Stripe        | `stripe.invoices.create`        | Yes                     | Voids the created invoice                         |
| Stripe        | `stripe.customers.create`       | Yes                     | Deletes the created customer                      |
| Slack         | `slack.chat.postMessage`        | Yes                     | Deletes the posted message                        |
| GitHub        | `github.issues.create`          | Yes                     | Closes the created issue                          |
