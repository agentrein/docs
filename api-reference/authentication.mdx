## SDK Authentication (API Key → JWT)

The SDK exchanges your organization API key for a short-lived JWT on first use.
```http
POST /auth/token
Content-Type: application/json

{ "apiKey": "ag_live_your_key_here" }
```

**Response**
```json
{ "data": { "token": "<jwt>" }, "error": null }
```

All subsequent SDK requests include this JWT:
```http
Authorization: Bearer <jwt>
```

## Dashboard Authentication

### Register
```http
POST /auth/register
Content-Type: application/json

{ "email": "user@example.com", "password": "secret", "name": "Ahmed" }
```

### Login
```http
POST /auth/login
Content-Type: application/json

{ "email": "user@example.com", "password": "secret" }
```

### OAuth

| Provider | Initiate | Callback |
|---|---|---|
| Google | `GET /auth/google` | `GET /auth/google/callback` |
| GitHub | `GET /auth/github` | `GET /auth/github/callback` |

OAuth callbacks redirect to the dashboard with a JWT in the URL query param `?token=`.

## Password Reset

### Forgot Password
```http
POST /auth/forgot-password
Content-Type: application/json

{ "email": "user@example.com" }
```

Always returns `200` regardless of whether the email exists, to prevent account enumeration.
```json
{ "message": "If this email exists, a reset link has been sent" }
```

If `RESEND_API_KEY` is configured, a reset link is emailed to the user. The link expires after 1 hour.

---

### Reset Password
```http
POST /auth/reset-password
Content-Type: application/json

{ "token": "uuid-reset-token", "newPassword": "newSecret" }
```
```json
{ "message": "Password reset successfully" }
```

Returns `400` if the token is invalid or expired.

## Error Format

All errors follow RFC 7807:
```json
{
  "data": null,
  "error": {
    "type": "https://agentrein.com/errors/unauthorized",
    "title": "Unauthorized",
    "status": 401,
    "detail": "Invalid API key"
  },
  "timestamp": "2025-01-01T00:00:00.000Z"
}
```
