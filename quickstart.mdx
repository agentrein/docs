---
title: Quickstart
description: Get AgentRein protecting your agent in under 5 minutes.
---

## 1. Install the SDK
```bash
npm install agentrein
```

## 2. Get Your API Key

Log in to [Dashboard](https://www.agentrein.com/sessions), go to **Settings**, and copy your API key.

## 3. Initialize the Client
```ts
import { AgentRein } from 'agentrein';

const agentrein = new AgentRein({
  apiKey: process.env.AGENTREIN_API_KEY,
});
```

## 4. Create a Session
```ts
const session = await agentrein.newSession({
  agentId: 'onboarding-agent',
  intent: 'Create a GitHub issue and notify the team on Slack',
});
```

## 5. Wrap Your Existing Clients
```ts
import { Octokit } from '@octokit/rest';
import { WebClient } from '@slack/web-api';

const octokit = new Octokit({ auth: process.env.GITHUB_TOKEN });
const slackClient = new WebClient(process.env.SLACK_TOKEN);

// Wrap once — use forever
const agentOctokit = agentrein.wrap(octokit, session, { connector: 'github' });
const agentSlack = agentrein.wrap(slackClient, session, { connector: 'slack' });
```

Your existing clients are unchanged. The wrapped versions look and behave identically — but every call is now tracked.

## 6. Use Them Exactly As Before
```ts
const issue = await agentOctokit.issues.create({
  owner: 'my-org',
  repo: 'my-repo',
  title: 'Onboarding: New Customer',
});

await agentSlack.chat.postMessage({
  channel: '#onboarding',
  text: `Issue created: ${issue.data.html_url}`,
});
```

If `chat.postMessage` throws, AgentRein automatically closes the GitHub issue. If `issues.create` throws, nothing needs to be reversed. The session always ends in a consistent state.

## 7. Complete the Session
```ts
await agentrein.completeSession(session);
```

<Info>
Need to protect an API not in the supported list? Use [`call()`](/sdk/call) with a custom [`undoConfig`](/sdk/undo-config) for full control over rollback logic.
</Info>
